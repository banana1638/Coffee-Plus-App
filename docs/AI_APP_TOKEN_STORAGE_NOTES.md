# AI App Token Storage Notes

## Goals

- Store auth token safely.
- Inject token consistently into API calls.
- Do not log token.
- Clear token on logout.
- Handle expired/invalid token gracefully.

## Review Areas

- token storage package
- auth service
- API client interceptor/header injection
- logout flow
- error handling for 401/403
- debug logs

## Detected Implementation

- Package: `flutter_secure_storage`.
- Storage key: `auth_token`.
- Primary implementation: lib/services/api_client.dart.
- Auth service: lib/services/auth_service.dart.
- Dio interceptor injects `Authorization: Bearer <token>` when available.
- 401 responses call `clearAuthState()`.
- API debug logs summarize token as present/none and should not log token values.

## Device Session Management

- Login and registration send `device_name` generated by `DeviceNameProvider`.
- `GET /tokens` is parsed as metadata only; plaintext token secrets are never expected or stored from this endpoint.
- Revoking the current token uses the backend `revoked_current` result and clears `flutter_secure_storage` through `ApiClient.clearAuthState()`.
- Revoking all tokens always clears local credentials.
- Revoking another device leaves the current local token intact.
