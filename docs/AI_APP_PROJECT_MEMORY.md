# AI App Project Memory

## Project Identity

Project:
- Coffee-Plus-App

Role:
- Flutter mobile application / API consumer

Paired backend:
- Coffee-Plus, a separate Laravel backend repository

## App Direction

Coffee-Plus-App should be:

- modern
- stable
- API-contract-aware
- environment-configurable
- safe with tokens
- clear in loading/error states
- visually polished like a premium cafe ordering app

## App Does Not Own Business Truth

The app displays and requests operations, but backend decides:

- product price
- coupon validity
- final order total
- payment success
- wallet balance
- order status
- user role
- ownership
